01 — Project Overview

The SEP (Systems Engineering Project) — LIMO Autonomous Navigation System involved designing, building, and validating a complete autonomous navigation stack for the AgileX LIMO differential-drive mobile robot using ROS 1 Melodic. The project covered the full pipeline: arena design, SLAM-based map building, localisation, path planning, and real-world obstacle avoidance.

Completed as a team, the module required the robot to autonomously navigate a custom-designed arena and respond to dynamic obstacles with reliable, repeatable performance.

02 — Objectives
  • 01Build a 2D occupancy grid map of a custom-designed arena using SLAM
  • 02Achieve reliable real-time pose estimation within the mapped environment
  • 03Plan and execute collision-free paths from start to goal poses using the ROS Navigation Stack
  • 04Validate reactive obstacle avoidance against static and dynamic obstacles in the arena

05 — System Architecture

SLAM

Simultaneous Localisation and Mapping using gmapping to generate a 2D occupancy grid map of the custom arena environment.

Localisation

Adaptive Monte Carlo Localisation (AMCL) for real-time pose estimation within the pre-built map during autonomous runs.

Path Planning

ROS Navigation Stack (move_base) with global planner (Dijkstra/A*) and local planner (DWA) for reactive obstacle avoidance.

Arena Design

Custom physical arena built to specification, with defined start/goal poses, static obstacles, and challenge zones.

06 — Technology Stack
  • Platform: AgileX LIMO differential-drive mobile robot
  • Middleware: ROS 1 Melodic on Ubuntu 18.04
  • Mapping: gmapping (particle filter SLAM)
  • Localisation: AMCL (Adaptive Monte Carlo Localisation)
  • Navigation: move_base · global_planner · DWA local planner
  • Visualisation: RViz
  • Languages: C++ (ROS nodes) · Bash (launch scripts)
07 — Role & Contribution

Within the team, I led the SLAM mapping phase — teleoperating the LIMO to scan the arena and producing the verified occupancy grid map — and tuned the AMCL localisation parameters for reliable pose estimation. I also worked on move_base costmap inflation and DWA local planner tuning to match the arena's geometry, and helped define and test waypoint goals in RViz during the autonomous validation runs.

08 — Verification & Validation

Map Verification

Saved and visually verified the occupancy grid map before progressing to the autonomous phase.

Localisation Tuning

Adjusted AMCL particle filter parameters and confirmed pose stability across repeated runs.

Planner Tuning

Tuned costmap inflation radii and DWA parameters for the specific arena geometry.

Obstacle Testing

Tested dynamic obstacle handling and recovery behaviour under the ROS Navigation Stack.

The LIMO robot successfully demonstrated fully autonomous navigation across the custom arena, including SLAM mapping, goal-directed path planning, and reactive obstacle avoidance — completing the Systems Engineering Project requirements with reliable multi-run performance.
